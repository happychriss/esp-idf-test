# HDC2080 Temperature + Humidity Sensor

## Overview

- Manufacturer: Texas Instruments
- Measures: temperature (–40 to +125 °C) and relative humidity (0–100 %RH)
- Interface: I2C
- Sleep current: 50 nA typical
- Default state after power-up: **sleep** (no measurement running)
- After single-shot measurement: automatically returns to sleep

## Connection — Waveshare ESP32-S3-Touch-LCD-1.9

The HDC2080 is an **external** add-on. It shares the I2C bus with the CST816S touch
controller and QMI8658A IMU.

| Signal     | GPIO | Notes                                      |
|------------|------|--------------------------------------------|
| SDA        | 47   | Shared with touch (CST816S) and IMU        |
| SCL        | 48   | Shared with touch (CST816S) and IMU        |
| ADDR       | —    | Tie to GND for 0x40, VDD for 0x41         |
| DRDY/INT   | —    | Optional; use polling if not wired         |
| VDD        | 3V3  |                                            |
| GND        | GND  |                                            |

### I2C Address

| ADDR pin | 7-bit address |
|----------|---------------|
| GND      | **0x40**      |
| VDD      | 0x41          |

Default in firmware: `0x40` (ADDR=GND). Change `HDC2080_I2C_ADDR` if wired differently.

## Key Registers

| Register | Address | Description                                  |
|----------|---------|----------------------------------------------|
| TEMP_L   | 0x00    | Temperature result LSB (bits 7:0)            |
| TEMP_H   | 0x01    | Temperature result MSB (bits 15:8)           |
| HUM_L    | 0x02    | Humidity result LSB                          |
| HUM_H    | 0x03    | Humidity result MSB                          |
| INT_DRDY | 0x04    | DRDY_STATUS bit7 — data ready flag (rd-clears)|
| MEAS_CFG | 0x0F    | Measurement configuration + trigger          |

### MEAS_CFG (0x0F) bit map

| Bits | Name      | Values                                       |
|------|-----------|----------------------------------------------|
| 0    | MEAS_TRIG | 1 = start measurement, auto-clears to 0     |
| 2:1  | MEAS_CONF | 00 = T+RH, 01 = T only                      |
| 4:3  | HRES      | 00 = 14-bit, 01 = 11-bit, 10 = 9-bit        |
| 6:5  | TRES      | 00 = 14-bit, 01 = 11-bit                     |

## Measurement Procedure (single-shot, polling)

```c
// 1. Trigger T+RH measurement, 14-bit resolution
uint8_t trig[2] = {0x0F, 0x01};
i2c_master_transmit(dev, trig, 2, 100);

// 2. Wait for conversion (14-bit T+RH ≤ 13 ms; use 15 ms margin)
vTaskDelay(pdMS_TO_TICKS(15));

// 3. Read TEMP_L then TEMP_H (auto-increments in single read)
uint8_t reg = 0x00;
uint8_t raw[2];
i2c_master_transmit(dev, &reg, 1, 100);
i2c_master_receive(dev, raw, 2, 100);

// 4. Convert
uint16_t raw16 = ((uint16_t)raw[1] << 8) | raw[0];
float temp_c   = ((float)raw16 / 65536.0f) * 165.0f - 40.5f;
```

## Deep Sleep

The HDC2080 is already in sleep after a single-shot measurement (AMM=0 default).
Before deep sleep, write 0x00 to MEAS_CFG (0x0F) to guarantee no pending trigger:

```c
uint8_t cmd[2] = {0x0F, 0x00};
i2c_master_transmit(dev, cmd, 2, 100);
```

No explicit power-down register is needed; 0x00 → MEAS_TRIG=0 → device stays asleep.

## I2C Bus Note

The HDC2080 sits on the same bus as CST816S and QMI8658A. Use the shared
`i2c_master_bus_handle_t`, add the device with `i2c_master_bus_add_device()`,
and remove it with `i2c_master_bus_rm_device()` after each transaction group.
