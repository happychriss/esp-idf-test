# QMI8658A — 6D IMU with Motion Co-Processor

Manufacturer: QST Corporation
Package: 2.5 × 3.0 × 0.86 mm, 14-pin LGA
Interface: I²C (standard/fast/fast+) or SPI
Operating voltage: VDD = VDDIO = 1.8 V (typical, per datasheet tables)

|-----------|---------|--------------------------------------------------|
| WHO_AM_I  | 0x00    | Returns 0x05 — confirms device identity          |

| CTRL8     | 0x09    | Motion routing + STATUSINT handshake mode        |

**WoM mode uses Low-Power accelerometer.** Our config (CTRL2=0x27 → 128 Hz LP) draws **~55 µA** — negligible for deep sleep budgets.

To further reduce WoM current: use 3 Hz LP (CTRL2[2:0]=100 is not valid for LP; lowest LP ODR is 3 Hz with CTRL2[2:0]=011... check Table 9). Practical minimum: 30 µA at 3 Hz. Response latency increases at lower ODR.

In accel-only mode, ODR is derived from the internal oscillator (not gyroscope).

|------------|------------------|---------------|---------|

| 011        | 250 Hz           | High-Res      | 142 µA  |

| 101        | 62.5 Hz          | High-Res      | 133 µA  |
| 110        | 31.25 Hz         | High-Res      | 132 µA  |

| 010 (LP)   | 11 Hz            | Low-Power     | 35 µA   |

CTRL2[5:3] = full-scale: 000=±2g, 001=±4g, 010=±8g, 011=±16g
Our config: CTRL2 = 0x27 → bits[5:3]=100 (±8g), bits[2:0]=111 (128Hz LP)

## Wake-on-Motion (WoM) — §12

WoM functions like Any-Motion detection but without buffering sensor data.
The IMU asserts INT1 (or INT2) when the acceleration slope exceeds the threshold.

### CAL1_H bit map

| Bits  | Function                                                         |
|-------|------------------------------------------------------------------|
| [7:6] | WoM interrupt pin + initial value: 00=INT1/LOW, 01=INT2/LOW, 10=INT1/HIGH, 11=INT2/HIGH |
| [5:0] | Interrupt Blanking Time (samples to ignore after WoM enable)     |

We use CAL1_H = 0x20: INT1 starts LOW → goes HIGH on motion → EXT1 wakes on ANY_HIGH.
32-sample blanking at 128 Hz ≈ 250 ms (screens startup transients).

### CAL1_L — WoM Threshold

Resolution: 1 mg/LSB. Range: 0x01–0xFF (0x00 = disable WoM).
Our value: **0xC0 = 192 mg** — clear shake or lift required.
More sensitive: 0x20 = 32 mg (trips on table vibration).

## Confirmed Working WoM Config Sequence

```c
imu_write(CTRL7, 0x00);         // disable all sensors
imu_write(CTRL1, 0x08);         // INT1 push-pull
imu_write(CTRL8, 0x80);         // STATUSINT handshake (avoids INT1 conflict during CTRL9)
imu_write(CTRL2, 0x27);         // ±8g, 128 Hz Low-Power
imu_write(CAL1_L, 0xC0);        // 192 mg threshold
imu_write(CAL1_H, 0x20);        // INT1 initial LOW, 32-sample blanking
imu_write(CTRL9, 0x08);         // CTRL_CMD_WRITE_WOM_SETTING
while (!(imu_read(STATUSINT) & 0x80)) {}  // poll CmdDone
imu_write(CTRL9, 0x00);         // host ACK
imu_write(CTRL7, 0x01);         // enable accelerometer
imu_read(STATUS1);               // clear any pending WoM flag
```

## EXT1 Wakeup (ESP32-S3 side)

```c
// GPIO8 must have RTC pulldown — floats HIGH without it → immediate false wakeup
rtc_gpio_init(GPIO_NUM_8);
rtc_gpio_set_direction(GPIO_NUM_8, RTC_GPIO_MODE_INPUT_ONLY);
rtc_gpio_pulldown_en(GPIO_NUM_8);
rtc_gpio_pullup_dis(GPIO_NUM_8);
esp_sleep_enable_ext1_wakeup(1ULL << 8, ESP_EXT1_WAKEUP_ANY_HIGH);
```

**Do NOT use `gpio_deep_sleep_hold_en()`** when EXT1 is needed — it latches GPIO8
LOW at sleep entry and blocks the wakeup edge. Use `rtc_gpio_hold_en()` only on
specific output pins that require holding.

## Notes

- IMU only configured when entering sleep — not needed at boot
- I2C address 0x6B confirmed working on this board (WHO_AM_I = 0x05)
- Low-Power WoM at 128 Hz = **55 µA** — NOT the source of 2 mA deep sleep issues
- Lowest LP ODR (3 Hz) reduces WoM current to 30 µA at cost of higher latency
